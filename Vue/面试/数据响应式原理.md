### 🔬 Vue响应式系统核心原理 (Vue 2 vs Vue 3)

```mermaid

A[数据变更] --> B(Vue响应系统)
B --> C[视图自动更新]
B --> D[计算属性/侦听器触发]
```

#### Vue 2 实现方案 (基于 Object.defineProperty)
```mermaid
flowchart TB
subgraph Vue2
  D[Data Object] --> E[Observer 观测]
  E --> F[递归遍历属性]
  F --> G[Object.defineProperty]
  G --> H[定义 Getter/Setter]
  H --> I[依赖收集 Dep]
  I --> J[通知 Watcher]
  J --> K[虚拟DOM更新]
end
```

**关键流程：**
1. **依赖收集**：在getter中通过`Dep.target`收集依赖（Watcher）
2. **拦截变更**：setter触发时通过`Dep.notify()`通知更新
3. **数组处理**：重写数组的7个变异方法（push/pop/shift/unshift/splice/sort/reverse）
4. **限制**：无法检测对象属性添加/删除

---

#### Vue 3 革命性改进 (基于 Proxy)
```mermaid
flowchart LR
subgraph Vue3
  A[响应式对象] --> B(Proxy代理)
  B --> C[handler.get]
  B --> D[handler.set]
  C --> E[track 追踪依赖]
  D --> F[trigger 触发更新]
  E & F --> G[ReactiveEffect]
  G --> H[调度更新]
end
```

**核心优势：**
```typescript
// 示例：Vue3响应式创建
const target = { count: 0 }
const proxy = new Proxy(target, {
  get(target, key) {
    track(target, key)  // 追踪依赖
    return target[key]
  },
  set(target, key, value) {
    target[key] = value
    trigger(target, key) // 触发更新
    return true
  }
})

// Ref实现 (基础类型)
function ref(value) {
  return {
    get value() {
      track(this, 'value')
      return value
    },
    set value(newVal) {
      value = newVal
      trigger(this, 'value')
    }
  }
}
```

### 🧠 深度响应式原理对照表
| 特性 | Vue 2 | Vue 3 |
|------|-------|-------|
| **核心机制** | Object.defineProperty | Proxy |
| **检测能力** | 现有属性修改 | 属性增/删/改/数组索引变更 |
| **性能** | 递归初始化代价高 | 按需代理，惰性初始化 |
| **嵌套处理** | 递归劫持所有嵌套对象 | 访问嵌套属性时才代理 |
| **数组支持** | 需覆盖7个方法 | 原生支持所有操作 |
| **生态影响** | Vue.set/Vue.delete特例 | 无需特殊API |
